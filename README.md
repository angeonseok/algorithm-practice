# Algorithm Practice

BOJ와 SWEA 문제 풀이를 정리하는 저장소입니다.

같은 문제를 Python과 C++로 모두 풀 수 있으므로, 언어별 분리가 아니라 플랫폼과 문제를 기준으로 정리합니다. 각 문제 폴더에는 풀이 코드와 문제별 `README.md`를 함께 둡니다.

## 디렉터리 구조

```text
algorithm-practice/
├── boj/
│   └── 카테고리/
│       └── 문제/
│           ├── 풀이 코드 (.py / .cpp)
│           └── README.md
└── swea/
    └── 난이도/
        └── 문제/
            ├── 풀이 코드 (.py / .cpp)
            └── README.md
```

## 정리 방식

- `boj`는 백준 단계별 분류를 기준으로 정리합니다.
- `boj/99.etc`는 단계별 분류에 넣기 애매한 문제를 난이도별로 모읍니다.
- `swea`는 `D1`~`D6`, `역량테스트`처럼 난이도와 유형 기준으로 정리합니다.
- 문제 폴더 안에는 풀이 파일과 `README.md`를 둡니다.
- 같은 문제를 여러 알고리즘으로 푼 경우에는 하나의 문제 폴더 안에 풀이 파일을 함께 둡니다.
  - 예: `문제번호.문제이름(방법1).py`, `문제번호.문제이름(방법2).py`

## 문제별 README

문제별 `README.md`는 문제 내용과 풀이를 다시 볼 때 필요한 내용을 정리합니다.

```markdown
# 문제번호.문제이름

## 문제 설명

## 제약사항

## 입력

## 출력

## 풀이 파일

- `문제번호.문제이름(방법1).py`: 첫 번째 알고리즘 풀이
- `문제번호.문제이름(방법2).py`: 두 번째 알고리즘 풀이

## 풀이 정리

### 핵심 아이디어

### 시간복잡도

### 구현 포인트

### 실수했던 점

### 다시 풀어볼 이유
```

`풀이 파일` 섹션은 같은 문제를 여러 방식으로 푼 경우에만 적어도 됩니다. 일반적으로는 문제 설명, 입력, 출력, 풀이 정리를 중심으로 작성합니다.

## 풀이 코드

- 온라인 저지에 바로 제출할 수 있는 단일 파일 형태로 작성합니다.
- 표준 입력과 표준 출력을 사용합니다.
- SWEA 문제는 보통 `#1 answer` 형식의 테스트케이스 출력 포맷을 따릅니다.
- BOJ 문제는 문제에서 요구하는 출력만 그대로 출력합니다.
